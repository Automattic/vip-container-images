Error processing the request

Server couldn't handle the response from PHP.

Troubleshooting steps

  • Check the server logs: 
    1. vip dev-env logs -f --slug my-app
    2. Repeat the action that caused the error message to appear.
    3. Check the log for the exact error message.
    4. Refer to common errors below.
    5. We recommend using XDebug to identify the problematic code.

Common errors

  • 502 upstream sent too big header while reading response header from upstream
    Application generated response headers that were too large for the server to handle. This can be caused by a large number of cookies or other headers.
  • 504 upstream timed out (110: Operation timed out)
    Application took too long to respond to the request. This can be caused by infinite loops, slow database queries, external remote requests, forgotten XDebug sessions, or other long-running operations.
If you're unable to resolve the issue please contact support.
